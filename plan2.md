# plan2

## 版本1

### 状态

失败 / 需要修正。

### 主要问题

1. 训练 prompt 与推理 prompt 不一致。
   - 训练侧使用的是：

   ```text
   问题：{question}
   答案：
   ```

   - 但推理 / eval 时，实际输入给服务的是原始 query，没有统一包装成相同模板。

2. answer 末尾没有显式 EOS。
   - 模型学到了如何开始回答，但没有充分学到“答案在何处结束”。

3. 短答案任务容易在答对后继续乱生成。
   - 典型现象：先输出正确短答案，随后继续打印无关文字，直到 `max_new_tokens`。

4. 结果对比不够完整。
   - 之前的 `Without memory` 不是严格意义上的 plain backbone 对照。

### 结论

版本1的主要失败原因不是数据本身变成了多选题，而是：

- 训练 / 推理格式不一致
- answer 结尾缺少 EOS
- 短答案边界监督不够清晰


## 版本2

### 当前修正

版本2的目标是先修正版本1里最基础的闭环问题。

### 已做修改

1. 训练与推理统一使用格式：

```text
问题：{question}
答案：
```

2. 训练目标补上 EOS：

```text
问题：{question}
答案：{answer}<eos>
```

3. eval 对比项补全为：

- `Gold / 标准答案`
- `With memory`
- `Without memory`
- `Direct Qwen`

### 版本2验证重点

1. 是否减少短答案答完后继续乱生成。
2. 是否让模型更稳定地在 EOS 处停止。
3. 是否让 `With memory / Without memory / Direct Qwen` 的对比更清晰。
4. 是否确认当前数据本身是直接答案任务，而不是多选任务。


## 版本3

### 计划方向

在版本2测试完成后，考虑将 prompt 从简单的 `问题 / 答案` 升级为更贴合 memory prefix 机制的弱指令模板。

推荐优先尝试：

```text
根据当前历史信息，回答下列问题。
请只输出最终答案，不要解释。
问题：{question}
回答：
```

训练目标对应为：

```text
根据当前历史信息，回答下列问题。
请只输出最终答案，不要解释。
问题：{question}
回答：{answer}<eos>
```

### 采用该方向的原因

1. 更符合当前真实机制。
   - memory 是 prefix embedding 注入，不是显式文本检索结果。

2. 比单纯 `问题：...\n答案：` 更明确提醒模型：
   - 前缀中的信息属于“当前历史信息”
   - 回答时应优先利用历史

3. `请只输出最终答案，不要解释。` 有助于抑制短答案任务中的额外续写。

### 版本3验证重点

1. 是否比版本2的简单模板更稳定。
2. 是否进一步减少答对后继续乱生成。
3. 是否提升地址、酒店名、城市、尾号等短答案任务的精确率。


## 版本4

### 计划方向

如果版本3仍不理想，再考虑更结构化的 prompt 变体，但不再使用 `thinking / 搜索结果` 这种容易虚构中间过程的形式，而是改为让模型先显式写出它抓到的“历史要点”，再给出最终答案。

候选示意：

```text
根据当前历史信息，先写出与问题最相关的历史要点，再给出最终答案。
问题：{question}
历史要点：
答案：
```

训练目标可对应为：

```text
根据当前历史信息，先写出与问题最相关的历史要点，再给出最终答案。
问题：{question}
历史要点：{relevant_fact_text}
答案：{answer}<eos>
```

### 当前判断

这个方向暂时不优先，但比原先的 `thinking / 搜索结果` 更贴合当前探索目标，原因是：

1. 当前系统虽然没有显式文本检索结果，但“历史要点”比“thinking”更像对 memory 中关键信息的文本化提炼。
2. 它更适合帮助分析模型是否真的抓到了正确历史事实，而不是只看最终答案对不对。
3. 相比 `thinking / 搜索结果`，它更少诱导模型去编造推理链。
4. 这个方向仍然比版本2、版本3复杂，因为它需要构造 `relevant_fact_text` 这类中间监督。

### 版本4适用条件

只有在版本2、版本3都不能明显改善短答案稳定性时，再进入这个方向验证。

并且需要先确认能否从现有数据中稳定构造出问题对应的 gold 历史要点监督。

### 版本4验证重点

1. “历史要点 + 答案”是否优于版本3的弱指令模板。
2. 模型输出的历史要点是否真的对应正确历史事实。
3. 是否有助于分析 memory 是否被真实利用。
4. 是否会引入新的虚构历史要点问题。
