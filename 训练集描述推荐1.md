# 针对 Stage 1 记忆追踪（Timeline / State Tracking）的数据集推荐

基于目前 `titans_lin` 中 `build_stage1_memory_data.py` 的数据生成逻辑，模型需要从一长串包含“实体属性覆写（Overwrite）”与“噪音干扰（Noop）”的历史记录中准确回答关于当前实体的最新状态。这种格式可以抽象为：
`history_chunks` + `question_chunk` -> `answer`

如果希望使用真实的开源数据集进行训练或评测，以下是几类高度契合的数据集推荐：

## 1. 任务型状态追踪对话：最契合的 overwrite 逻辑
**推荐数据集：[MultiWOZ (2.1 / 2.4)](https://github.com/budzianowski/multiwoz)**
* **简介**：最经典的跨领域任务型对话数据集。用户会在多轮对话中修改自己的需求（比如一开始要求市中心，后来改成偏远地区）。
* **封装方式**：
  * `history_chunks`：放入前面每一轮的用户和系统对话内容。
  * `question_chunk`：针对当前的对话状态提问（例如："What area is the hotel the user looking for?"）。
  * `answer`：当前槽位（Slot）的最新值。
* **契合度评估**：**极高**。完全符合在漫长对话中“实体属性覆盖更新”以及抓取最新有效信息的核心逻辑。

## 2. 长程人设和多回话记忆追踪
**推荐数据集：[MSC (Multi-Session Chat)](https://parl.ai/projects/msc/) / PersonaChat**
* **简介**：Facebook 构建的长程多轮对话数据集。对话跨越多个 Session，涉及职业、爱好等长程信息的建立和更新。
* **封装方式**：
  * `history_chunks`：将前面的数个 Session 切片打平放入。
  * `question_chunk`：针对这些长期记忆的事实提问。
  * `answer`：从最新历史中提取的短答案。
* **契合度评估**：**高**。非常符合“用户档案（Profile）/长效社交属性”的记忆更新机制。

## 3. 长文本阅读与多跳检索（长上下文能力底座）
**推荐数据集：HotpotQA, Qasper 或直接使用 LongBench 中的数据集**
* **简介**：标准的阅读理解与多跳推理，模型从极长文本中寻找出答案。
* **封装方式**：
  * `history_chunks`：维基百科或论文按段落切分。
  * `question_chunk`：原生多跳问题。
  * `answer`：原生短答案。
* **契合度评估**：中等（缺乏动态覆盖逻辑）。但它是保障模型“大海捞针（NIAH）”寻回能力的极佳补充。

## 4. 纯逻辑时间线与推导
**推荐数据集：bAbI (Task 1-20) / BABILong**
* **简介**：纯粹的合成时间线与逻辑推理跟踪（如实体位置、持有物品的转移）。
* **封装方式**：
  * `history_chunks`：环境状态句子（可穿插随机维基百科词条作为混淆背景）。
  * `question_chunk` & `answer`：直接使用 bAbI 设定好的逻辑问答。
* **契合度评估**：**高**。并且可以轻松合成拓展至 128K 以上的长度做极端压力测试。

## 5. 常规指令防止“灾难性遗忘”
**推荐数据集：ShareGPT / UltraChat**
* **简介**：多领域的真实用户和 AI 的多轮指令对话。
* **封装方式**：
  * `history_chunks`：前 N-1 轮对话。
  * `question_chunk`：第 N 轮的用户指令。
  * `answer`：AI 的对应回答。
* **补充说明**：仅使用前面4种高度垂直的数据可能导致模型正常对话能力衰退，混入通用多轮指令（10%左右）可以维持其指令遵循和回复语气。

### 数据集转换示例 (以 MultiWOZ 为例)
建议编写一个类似于 `convert_multiwoz_to_stage1.py` 的脚本，将最新状态转换为平铺格式（Flat Format）供训练：

```json
{
  "episode_id": "multiwoz_001_turn3",
  "history_chunks": [
    "User: I need a hotel in the centre.", 
    "Sys: Any price range?", 
    "User: Cheap please."
  ],
  "question_chunk": "What price range is the user looking for?",
  "answer": "cheap"
}
```
这样的 JSONL 格式可以直接喂入 `Stage1Dataset` 管道。
